Бизнес-допущения и зависимости​

Критические допущения:

  • Все микросервисы используют или могут быть переведены на OpenAPI 3.0 для документирования API.
  • Команды разработки готовы выделять 2–3 часа в неделю на регистрацию сервисов в первые 3 месяца.
  • Руководство поддерживает инициативу и выделяет ресурсы DevOps для технической интеграции.
  • Существующая инфраструктура поддерживает развёртывание новых сервисов (Kubernetes-кластер с доступными ресурсами).
  • Средняя частота изменений API: 2–3 критических изменения на сервис в год.

Внешние зависимости:

  • Библиотеки Open Source: oasdiff (MIT), react-flow (MIT), PostgreSQL.
  • Облачная инфраструктура: Docker для контейнеризации, Kubernetes для оркестрации.
  • Существующие системы: GitHub/GitLab для хранения кода, PagerDuty для управления инцидентами.
